# 角色分级与权限矩阵

来源：`I:\software Project\乳腺影像与临床数据辅助分析系统_需求规格说明书(2).docx`

## 角色边界

| 角色 | 职责定位 | 可进入页面 | 允许的写操作 | 禁止事项 |
|---|---|---|---|---|
| 医生用户 | 一线业务使用者 | 仪表盘、患者管理、影像与ROI、分析任务 | 创建/修改患者，录入临床病理，上传MRI影像，保存ROI标注，发起pCR辅助分析，导出单任务报告 | 不能审核数据，不能管理账号 |
| 科室管理员 | 数据质控与科室统计管理 | 仪表盘、患者管理、影像与ROI、分析任务、审核管理 | 审核医生提交的数据，填写通过/退回意见 | 不能代替医生录入患者、上传影像、保存ROI或发起分析任务 |
| 系统管理员 | 平台账号与权限运维 | 仪表盘、用户管理 | 创建用户、分配医生/科室管理员/系统管理员角色 | 不参与患者业务数据、影像标注、pCR分析或科室审核 |

## 当前落地方式

| 层级 | 实现 |
|---|---|
| 前端路由 | `frontend/src/rolePolicy.ts` 统一定义可访问页面，未授权跳转 `/403` |
| 前端按钮 | 医生可见新增、上传、保存ROI、创建分析、导出PDF；科室管理员进入只读质控模式 |
| 后端接口 | 患者/临床/影像/标注/分析读接口限定医生与科室管理员；业务写接口只允许医生；审核接口只允许科室管理员；用户管理只允许系统管理员 |
| 影像来源 | 后端返回 `source_type`，前端显示本地验证数据集MRI影像、医生上传影像或模拟样例影像 |

## 数据集说明

影像管理模块当前优先使用 `I:\software Project\数据集` 导入的本地验证数据集 PNG 影像。合成数据只用于补足课程训练演示，公开 Wisconsin 数据只作为表格分类教学基准，不混入真实 MRI/pCR 结论。
