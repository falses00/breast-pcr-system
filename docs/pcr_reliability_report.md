# pCR辅助分析与可靠性说明

## 结论

当前 pCR 结果只能作为课程项目中的辅助分析展示，不能认为可靠到可用于真实临床决策。系统可靠的是工程链路：数据导入、ROI 标注、特征提取、规则模型、机器学习训练、指标输出和权限审核；不可靠的是临床泛化性能。

## 当前数据

- 本地验证集：35例临床记录、35张 PNG 影像。
- 当前可用于 pCR 训练评估的有标签样本：33例。
- 公开 Wisconsin 数据：569行表格诊断数据，只用于机器学习流程教学基准，不用于 pCR 或 MRI。
- 合成教学数据：默认120例，根据本地字段分布和灰度统计推测生成，只用于课程流程演示。

## 已使用机器学习

| 模型 | 输入 | 用途 |
|---|---|---|
| 规则评分模型 | 年龄、ER、PR、HER2、Ki-67、病灶面积、灰度均值 | pCR 辅助概率演示 |
| Logistic Regression | 本地临床特征 | Clinical-only pCR 演示 |
| Random Forest | 本地整图灰度特征 | Image/Radiomics-only 演示 |
| Random Forest | 本地临床 + 影像灰度特征 | 融合模型演示 |
| Gradient Boosting | 本地临床 + 影像灰度特征 | XGBoost路线轻量替代演示 |
| Logistic Regression / Random Forest | Wisconsin公开表格数据 | 良恶性分类教学基准，非 pCR |
| Logistic Regression / Random Forest / Gradient Boosting | 本地样例 + 合成教学数据 | 样本不足时的课程增强演示，非真实临床性能 |

## 深度学习状态

当前未训练深度学习模型。原因：

- 35例样本不足以训练 2.5D/3D CNN。
- 当前影像是 PNG 截图/样例，缺少完整 DCE-MRI 体数据、时相、序列和严格 ROI。
- 真实深度学习路线需要 DICOM/NIfTI、患者级划分、外部验证和医学专家审核。

已预留路线：

- 2.5D ResNet / 3D ResNet。
- MONAI / TorchIO 数据管线。
- nnU-Net / Swin UNETR 扩展。
- 多模态融合：影像编码器 + 临床 MLP + 注意力融合。

## 是否需要接入大模型

MVP 不需要大模型参与预测。建议只把 GPT-5.5 或其他大模型接入为“报告生成助手”：

- 本地规则模型或机器学习模型先输出概率、特征和解释因子。
- 大模型只把结构化结果整理成中文说明。
- 大模型不得直接看 MRI 后给诊断。
- 大模型报告必须包含免责声明。

## 影响分析

当前系统对课程项目的价值：

- 能展示完整医学 AI 工程链路，而不是单独模型脚本。
- 能解释 pCR 相关因素如何进入规则模型和传统机器学习模型。
- 能体现数据治理、权限管理、审核流程和医学免责声明。

当前系统对真实临床的限制：

- 样本太少，无法支撑稳定 AUC、召回率、校准度。
- PNG 影像不能代表完整 DCE-MRI。
- 未做外部验证、前瞻性验证和临床读片对照。
- 合成数据可改善演示流程丰富度，但会引入人为规则偏差，不能提升真实临床证据等级。
- 因此输出必须定位为课程展示，不可声称诊断能力。
